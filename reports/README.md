# reports/

Pipeline runs drop their narrative outputs here:

* `module{N}_report.md` — one per module: what was reproduced, deviations from the published result, hypotheses for discrepancies.
* `data_manifest.json` — SHA-256 checksums + source URLs for every downloaded file (written automatically by the download scripts).

These reports are the lab notebook of the project — keep them under version control (except the machine-generated manifest).
